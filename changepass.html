Change Password User Guide

This guide explains how to use the **Change Password** function within the **HK.Huang Sales Management Software**. Regularly changing your password is a critical security practice to protect your account and sensitive data.

Understanding the Change Password Screen

The Change Password screen allows authenticated users to update their login password. It typically consists of three input fields and action buttons.

Image: The Change Password interface showing input fields and buttons.

How to Change Your Password

To change your password, follow these steps:

1. Access the Change Password Function

Ensure you are logged into the application. The "Change Password" option is usually found in the main application menu (e.g., under "Account" or "File" menu), or a dedicated toolbar button (refer to mnuChangePass_Click method in frmMain.cs).

2. Fill in the Password Fields

  • Old Password: Enter your current password in this field (txtOldPass).
  • New Password: Enter your desired new password in this field (txtNewPass).
  • Confirm New Password: Re-enter your new password in this field to confirm it (txtConfirm). This helps prevent typing errors.
Password Security Tips:
Choose a strong password that is at least 8 characters long, includes a mix of uppercase and lowercase letters, numbers, and symbols. Avoid using easily guessable information like birthdays or common words.

3. Submit Your New Password (btnChange_Click)

After filling in all fields, click the Change button.

The system will perform several validations:

  • All Fields Required: If any field is left empty, a warning message "Please fill in all fields!" will appear.
  • New Passwords Must Match: If the "New Password" and "Confirm New Password" do not match, an error message "New passwords do not match!" will appear.
  • Correct Old Password: The system will verify if the "Old Password" you entered is correct. If it's incorrect, an error message "Old password is incorrect!" will appear.
  • User Not Found: In rare cases, if the user account cannot be identified, an error "User not found!" might be shown.

If all validations pass successfully, a "Password changed successfully!" message will be displayed, and the Change Password window will close (as per btnChange_Click logic).

4. Cancel Operation (btnCancel_Click)

If you decide not to change your password, click the Cancel button. This will close the Change Password window without making any modifications (as per btnCancel_Click logic).

Note:
For security purposes, it is recommended to remember your new password immediately. If you forget it after changing, you will need to contact the system administrator for assistance.
