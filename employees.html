Employee Management User Guide

This guide provides instructions for managing employee information within the **HK.Huang Sales Management Software**. Effective employee management is crucial for operational efficiency and security.

Overview of the Employees Screen

The Employees management screen (frmEmployees) allows you to view, add, edit, delete, import, and export employee details. The main components are:

  • Employee List (Data Grid View): Displays all existing employees with their key details including ID, Full Name, Username, Address, Phone, and Role.
  • Employee Details Panel: Shows the details of the currently selected employee for editing or new entry. This includes Full Name, Username, Password, Address, Phone, and Role.
  • Action Buttons: Buttons for various operations like Add, Update, Delete, Save, Cancel, Import, and Export.
  • Navigation and Search Bar: Tools to navigate through the list and find specific employees.

Viewing and Navigating Employees

When you open the Employees screen (Employees_Load), the system automatically loads all available employee records into the data grid view (dataGridView).

1. Navigating Through Records

Use the navigation buttons on the toolbar to move through the list of employees:

  • First (btnBegin): Moves to the first record.
  • Previous (btnPrevious): Moves to the previous record.
  • Next (btnNext): Moves to the next record.
  • Last (btnEnd): Moves to the last record.

The details of the currently selected employee will be displayed in the respective fields: "Full Name" (txtEmployeeName), "Username" (txtEmployeeUsername), "Address" (txtEmployeeAddress), "Phone" (txtEmployeePhone), and "Role" (cboRoles).

Note: For security reasons, the "Password" field (txtEmployeePassword) will be cleared when controls are enabled for editing or adding, and it does not display the existing password. You will need to enter a new password when updating or adding an employee.

2. Searching for Employees

To find a specific employee:

  • Enter keywords into the search box (txtFind).
  • As you type, the list will automatically filter to show employees whose Full Name, Address, Phone, or Username contain the entered keyword (triggered by txtFind.TextChanged calling btnFind.PerformClick).
  • If no matching employees are found, a "No matching customer found." (likely means "No matching employee found.") message will appear (in lblMessage).
  • Click the Clear button (btnClear_Click) to clear the search filter and display all employees again.

Adding a New Employee

To add a new employee record:

  1. Click the Add button (btnAdd_Click). All employee detail fields will become active and clear for new input, and the "Save" and "Cancel" buttons will be enabled. The cursor will focus on the "Full Name" field.
  2. Enter the details for the new employee:
    • Full Name (txtEmployeeName)
    • Username (txtEmployeeUsername)
    • Password (txtEmployeePassword) - *Required for new employees.*
    • Employee Address (txtEmployeeAddress)
    • Employee Phone (txtEmployeePhone)
    • Role (cboRoles) - Select the employee's role (e.g., Administrator or Staff).
  3. Click the Save button (btnSave_Click) to create the new employee.
  4. A "Operation successful!" message will confirm the addition. The screen will then refresh, and the new employee will appear in the list.
Note: Ensure all required fields are filled accurately, especially Username and Password for login.

Editing an Existing Employee

To modify an existing employee record:

  1. Select the employee you wish to edit from the data grid view (dataGridView) by clicking on their row. The employee's details will appear in the input fields.
  2. Click the Update button (btnUpdate_Click). All employee detail fields will become editable, and the "Save" and "Cancel" buttons will be enabled. The password field will be cleared.
  3. Modify the employee details as needed in the txtEmployeeName, txtEmployeeUsername, txtEmployeeAddress, txtEmployeePhone, and cboRoles fields.
  4. To change the password: Enter the new password in the txtEmployeePassword field. If you do not enter a new password, the existing one will be retained (assuming the system handles this, as the field is cleared).
  5. Click the Save button (btnSave_Click) to apply your changes.
  6. A "Operation successful!" message will confirm the update. The screen will then refresh to reflect the changes.

Deleting an Employee

To remove an employee record:

  1. Select the employee you wish to delete from the data grid view (dataGridView).
  2. Click the Delete button (btnDelete_Click).
  3. A confirmation message "Are you sure you want to delete this employee?" will appear. Click Yes to proceed or No to cancel.
  4. If confirmed, an "Employee deleted successfully!" message will appear, and the employee will be removed from the list.
Warning: Deleting an employee will permanently remove their record from the system. Ensure the employee no longer requires access.

Importing Employees from Excel

You can import multiple employee records from an Excel file:

  1. Click the Import button (btnImport_Click). An "Open File" dialog will appear.
  2. Navigate to and select your Excel file (e.g., .xls or .xlsx).
  3. The system will read the data from the first worksheet. Ensure your Excel file has header columns named exactly: "FullName", "UserName", "Password", "EmployeeAddress", "EmployeePhone", and "Role". The "Role" column should contain boolean values (e.g., TRUE/FALSE or 1/0) or equivalent text that can be converted to boolean (e.g., "True" for Administrator, "False" for Staff).
  4. After the import process, a message indicating how many rows were successfully imported will appear.
Excel File Format:
The Excel file must have a header row with the exact column names: "FullName", "UserName", "Password", "EmployeeAddress", "EmployeePhone", and "Role". Ensure "Role" column data is compatible for boolean conversion.

Exporting Employees to Excel

You can export the current list of employees to an Excel file:

  1. Click the Export button (btnExport_Click). A "Save File" dialog will appear.
  2. Choose a location and enter a file name for your Excel file (e.g., Employees_YYYY_MM_DD.xlsx).
  3. Click Save.
  4. A "Exported data to Excel file successfully." message will confirm the export.
  5. The Excel file will contain columns for ID, Full Name, Employee Phone, Employee Address, Username, and Role.

Cancelling Actions and Closing

  • Cancel Button (btnCancel_Click): If you are in "Add" or "Update" mode, clicking "Cancel" will discard any unsaved changes and return the screen to its initial state, reloading the original data.
  • Close Button (btnClose_Click): Closes the Employees management screen and returns you to the main application.
